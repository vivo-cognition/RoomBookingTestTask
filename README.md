# Тестовое задание: API для системы бронирования переговорных комнат

RESTful API для внутренней системы бронирования переговорных комнат в офисе с использованием ASP.NET Core и EF Core. Проект выполнен в рамках тестового задания.

Технологический стек
•	Платформа: .NET 10
•	Фреймворк: ASP.NET Core Web API
•	ORM: Entity Framework Core
•	СУБД: SQLite или PostgreSQL (на твой выбор)
•	Документация: Swagger (OpenAPI) был использован Scalar

Архитектурные особенности и фичи
  - Clean-подход: Логика API полностью изолирована от сущностей БД через паттерн DTO (Data Transfer Objects).
  - Авто-миграции: База данных разворачивается и накатывает схемы таблиц автоматически при первом запуске приложения (настроен таймаут на ожидание готовности PostgreSQL).
  - Бизнес-валидация: Реализованы жесткие проверки пересечения временных интервалов, валидация длительности брони (от 15 мин до 8 часов), уникальность названий комнат и каскадное удаление зависимых броней.
  - Логирование: Интегрировано базовое логирование ключевых бизнес-действий (создание, отмена броней) и системных предупреждений через встроенный ILogger.

Начиная с .NET 9/10, Microsoft полностью удалила стороннюю библиотеку Swashbuckle из стандартных шаблонов. Вместо устаревшего Swagger UI в данном проекте развернут современный интерактивный инструмент Scalar, работающий напрямую со встроенным генератором Microsoft.AspNetCore.OpenAPI

Проект полностью упакован в Docker-контейнеры. Для запуска вам понадобится только установленный Docker Desktop.
Выполните команду в корне проекта:
```bash
docker-compose up --build
```

После успешного запуска документация и интерактивная панель для тестирования запросов будут доступны по адресу:
  - Scalar UI: `https://localhost:8081/scalar` (или через проброшенный порт в Docker Desktop)
  - Сырой OpenAPI JSON: `https://localhost:8081/openapi/v1.json`
